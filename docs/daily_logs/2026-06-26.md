### What I learned
- Variance from first principles: covariance is safe for output (substitute toward a more specific type), contravariance is safe for input (substitute toward a more general type), and a type used as both (a mutable container) must be invariant. Functions split these (parameters contravariant, return covariant), which is why they are not invariant.
- super().__init__(message) on a custom exception runs the base Exception constructor, which stores the message in args so str(exc) and tracebacks show it. Defining your own __init__ overrides the base, so the call is needed to keep the standard printable-message behavior. This serves the developer (logs, debugging), separate from the custom attributes that serve the HTTP client.
- response_model documents only what is declared; the default validation-error example in the docs was FastAPI's, not mine, until responses= told the schema about the real error shape. The runtime behavior was already correct; responses= makes the documentation honest.
- docker-compose up handles both build and run when the service declares build:, replacing the separate docker build and docker run commands with one declarative file.
