### What I learned
- Pickle deserialization in depth: pickle stores class references plus data, and on load imports the class from the installed library and pours the saved parameters into a blank instance. No retraining. This is why scikit-learn must be installed and version-matched.
- The Request object: built by FastAPI from the ASGI scope after arrival, wraps the still-raw body bytes (parsed lazily), and carries a pointer to the app (request.app) which is what lets a dependency reach app.state without importing app.
- Dependency injection is resolved by FastAPI only at the route boundary. Calling a function directly does not resolve Depends(); the dependency must be injected at the route and passed down as a normal argument.
- Two error-response strategies: return the error model per-route (Strategy A, defaults to 200, duplicated) versus raise typed exceptions and handle them globally (Strategy B, clean status codes, centralized). Chose B.
- response_model does not run on a JSONResponse or on handler responses, so handlers build ErrorResponse manually to guarantee the shape. The success path gets automatic shape-enforcement; the error path gets manual shape-enforcement.
