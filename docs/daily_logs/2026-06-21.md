### What I learned
- Containers share the host kernel instead of virtualizing hardware like a VM, which is why they are lightweight. A Linux container needs a Linux kernel, supplied directly on Linux hosts and via a small hidden VM on Mac/Windows.
- Docker layer caching: each instruction is a cached layer, rebuilds happen top to bottom, and once one layer changes every layer below it rebuilds. For COPY, the file contents are checksummed, so copy order controls whether dependencies reinstall on every code change.
- The two gates for reaching a containerized server: --host 0.0.0.0 makes uvicorn listen on all interfaces inside the container, and -p host:container publishes a port through the container's network boundary. Both are required.
- uv in Docker: copy the prebuilt binary from the uv image rather than running the installer, and pin the version for reproducibility.
